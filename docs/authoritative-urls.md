## Citation Format Examples

### GDPR with Official URL
```yaml
citations:
  - regulation: GDPR
    article: 4(1)
    description: Personal data includes any identifiable information
    url: https://eur-lex.europa.eu/eli/reg/2016/679/oj
```

### HIPAA with CFR Reference
```yaml
citations:
  - regulation: HIPAA
    section: 45 CFR ยง164.514(b)(2)(i)(A)
    url: https://www.ecfr.gov/current/title-45/subtitle-A/subchapter-C/part-164
```

### LGPD with Portuguese Official Text
```yaml
citations:
  - regulation: LGPD
    article: Article 5, I
    description: Definition of personal data
    url: http://www.planalto.gov.br/ccivil_03/_ato2015-2018/2018/lei/l13709.htm
```

### National Law with Authority Guidance
```yaml
citations:
  - regulation: GDPR
    article: 4(1)
    url: https://eur-lex.europa.eu/eli/reg/2016/679/oj
  - national_law: Bundesdatenschutzgesetz (BDSG)
    section: Section 26
    description: Employee data requires special protection
    url: https://www.gesetze-im-internet.de/bdsg_2018/
```

## URL Validation Checklist

Before adding a URL to a citation:

- [ ] URL is from an official government or regulatory source
- [ ] URL is accessible (not behind paywall or login)
- [ ] URL is stable (permalink preferred over dynamic URLs)
- [ ] URL points to the correct article/section when possible
- [ ] URL is properly formatted (https:// preferred)
- [ ] For non-English laws, consider adding English translation URL

---

## Maintenance Notes

- **Update Frequency**: Review URLs quarterly for broken links
- **Validation**: Run automated link checker on all URLs
- **Alternatives**: Maintain backup URLs (e.g., archive.org) for critical references
- **Translations**: Prioritize official translations over third-party
